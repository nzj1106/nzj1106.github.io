hexo免费搭建一个属于自己的博客

Veröffentlicht am 2018-12-27 | in Web技术 . 生活琐事

今天看到一篇写的不错的hexo搭建博客教程
下面这个就是网址：
https://www.jianshu.com/p/51617690f8ca

我的搭建教程之后有时间再写上

markdown 文档网址，之后写文章需要用到

http://www.markdown.cn/

一、前言：

该博客的搭建用的是Hexo框架，基于Node.js,搭建博客首先需要有一个github账号并且配置了SSH Keys,然后在github上面创建一个名为username.github.io的仓库，
还需要安装Git、Node、hexo

安装环境稍后补上，现在直接从hexo开始写

二、准备工作

1、安装Hexo

等所有必备的应用程序安装完成后，即可用npm安装hexo

npm install -g hexo-cli

三、建站

1、创建资源

安装完hexo之后，在你喜欢的目录下面新建一个文件夹命名为 username（替换成你喜欢的名字，例如：nzj1106），然后cd到username文件夹下

$ cd username
$ hexo init
$ npm install

执行完毕后你将看到你刚创建的文件夹下会有很多资源，如下图：

2、基础配置

打开文件夹下的 _config.yml 文件，修改一下里面的几个关键信息，记得保存，注意：冒号后面要有空格
下图是我个人的配置情况,如需修改其他信息请参考 配置

3、主题安装

themes文件夹中方的是主题，默认是landscape,如果不喜欢，可以更换其他主题。例如：next

  • next 主题安装

    $ cd username  //进入到你创建的文件夹下
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

  • 启用next主题

    打开配置文件_config.yml，找到themes字段将其改为next

  • 验证主题：（注意：每次切换主题后，验证主题之前都要清除hexo的缓存）

    $ hexo clean //清除缓存

  • 开始验证：启动hexo本地服务器

    $ hexo s

然后你将会在终端中输出

INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

这行的意思是说你用浏览器打开 http://localhost:4000 便可看到你的博客了，这只是本地的测试，如果测试没问题就执行Ctrl+C退出去，此时该地址就不能访问了。

四、写作

以上所有的工作都完成了，并设置了一套你自己喜欢的主题样式，接下来就可以写博客了。

1、新建一篇文章

$ hexo new layout title

layout：模板，如果不写将使用默认模板，一般不写，直接$ hexo new title
title：文章标题，如果标题包含空格的话，请使用引号括起来。
新建的文章将会存在username/source/_posts中，_posts就是默认模板，文章文件名称将会是title.md，接下来就可以打开该文件进行编辑写你的文章了。

2、生成静态文件

写完文章之后执行，来生成静态网页文件
$ hexo g //hexo generate的缩写

3、启动本地服务器
$ hexo s        //hexo server的缩写

本地服务器启动完之后就如同上面验证主题一样，访问 http://localhost:4000 来验证一下你写的文章预览效果。

4、部署网站

安装部署工具
$ npm install hexo-deployer-git –save

安装完毕后打开username下的_config.yml文件，翻到最底部找到deploy节点编辑如下：

deploy:
type: git
repository: https://github.com/YoungerLi/youngerli.github.io.git
branch: master

其中https://github.com/YoungerLi/youngerli.github.io.git就是你GitHub中创建的仓库地址，在这复制即可

最后，部署网站
$ hexo d //hexo deploy的缩写

第一次执行这一步的时候会让你输入你的GitHub的用户名及密码，输入正确无误后执行完毕，任何人就可以访问 https://youngerli.github.io 进入你的博客了。

以后每次写完文章之后依次执行以下命令就可以发布更新了

小插曲
使用markdown语法插入图片的时候遇到个小问题，不知道怎么插入本地图片，百度了一下，图片成功显示

首先需要在source文件夹中新建一个images文件夹，把图片放在images文件夹里，路径如下：

![alt text](../../images/img.jpg)

不能在网页中使用绝对路径，如果想部署到服务器上，必定是相对路径。
markdown本来的语法中，只要img和md文件的相对路径是对的就行。
